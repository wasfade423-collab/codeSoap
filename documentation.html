1. Introduction

Cette API SOAP constitue le canal d'intégration synchrone pour les partenaires externes du Port Autonome de Cotonou. Développée intégralement en Python 3 à l'aide de la bibliothèque standard, elle garantit une empreinte mémoire minimale et une exécution hautement sécurisée sans framework tiers.

2. Architecture du code

Le projet est structuré de manière modulaire autour de six composants applicatifs fondamentaux :

  • config.py : Centralisation des configurations de variables d'environnement.
  • db.py : Gestionnaire de l'instance de connexion au SGBD MySQL.
  • server.py : Serveur d'écoute HTTP de bas niveau et aiguillage des requêtes.
  • services.py : Analyseur XML de payloads et exécution des requêtes métiers SQL.
  • templates.py : Schémas de structures XML et dictionnaire WSDL.
  • requirements.txt : Manifeste de dépendance unique pour le pilote de base de données.

3. Endpoints & Services

Service : Consultation de Dossier (GetDossier)

Extrait les détails d'un dossier de fret à partir de sa référence unique.

Méthode HTTP: POST
URL Endpoint: http://localhost:8000/
Header 1: Content-Type: text/xml; charset=utf-8
Header 2: SOAPAction: GetDossier

Structure de la Requête XML 

<?xml version="1.0" encoding="UTF-8"?>
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:smart="http://smartport.lacotonou.bj/">
   <soapenv:Header/>
   <soapenv:Body>
      <smart:GetDossierRequest>
         <reference_metier>DOS-2026-XYZ</reference_metier>
      </smart:GetDossierRequest>
   </soapenv:Body>
</soapenv:Envelope>

Structure de la Réponse XML (Succès) 

<?xml version="1.0" encoding="UTF-8"?>
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:smart="http://smartport.lacotonou.bj/">
   <soapenv:Header/>
   <soapenv:Body>
      <smart:GetDossierResponse>
         <id>a1b2c3d4-e5f6-7a8b-9c0d-1e2f3a4b5c6d</id>
         <statut>valide</statut>
         <type_operation>import</type_operation>
      </smart:GetDossierResponse>
   </soapenv:Body>
</soapenv:Envelope>

4. Sécurité & Audit

L'architecture logicielle applique les préceptes du modèle Zero-Trust. L'analyse syntaxique du XML utilise de façon restrictive le module xml.etree.ElementTree natif, configuré intrinsèquement pour rejeter les résolutions d'entités externes XML (XXE Vulnerabilities). Toute action initiée via le canal SOAP consigne l'empreinte de la transaction de manière séquentielle et immuable au sein du journal d'audit de la base de données.
