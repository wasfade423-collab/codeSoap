🚢 SmartPort Gateway

Documentation officielle de l'API SOAP

Version 1.0

📌 Présentation

SmartPort Gateway est une API SOAP développée en Python permettant la gestion et le suivi des dossiers de transit portuaire. Elle fournit des services SOAP standardisés pour créer, consulter et suivre des dossiers métier.

Les données sont stockées dans une base PostgreSQL hébergée sur Render.

🌐 URL du Service

Le service SOAP est accessible à l'adresse suivante :

https://codesoap.onrender.com/

📄 Contrat WSDL

Le contrat WSDL peut être consulté directement via :

https://codesoap.onrender.com/?wsdl
Consulter le WSDL

🚀 Fonctionnalités

  • Création de dossiers de transit
  • Consultation de dossiers
  • Suivi de statut
  • Gestion SOAP XML
  • Génération automatique du WSDL
  • Initialisation automatique de la base de données

🏗️ Architecture du Projet

Fichier Description
server.py Gestion des requêtes HTTP et SOAP
services.py Logique métier et traitement XML
db.py Connexion PostgreSQL et requêtes SQL
templates.py Réponses SOAP et définition du WSDL

🗄️ Structure de la Base de Données

CREATE TABLE dossiers (
    id SERIAL PRIMARY KEY,
    reference_metier VARCHAR(255) NOT NULL UNIQUE,
    statut VARCHAR(50) NOT NULL,
    type_operation VARCHAR(100) NOT NULL
);
Champ Description
id Identifiant unique
reference_metier Référence métier du dossier
statut Statut actuel du dossier
type_operation Type d'opération portuaire

🔗 Services SOAP Disponibles

CreateDossier

Création d'un nouveau dossier.

Paramètre Type
reference_metier String
statut String
type_operation String

GetDossier

Consultation d'un dossier existant.

Paramètre Type
reference_metier String

📬 Exemple de Requête SOAP

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/">
   <soapenv:Body>
      <CreateDossierRequest>
         <reference_metier>REF001</reference_metier>
         <statut>EN_COURS</statut>
         <type_operation>IMPORT</type_operation>
      </CreateDossierRequest>
   </soapenv:Body>
</soapenv:Envelope>

📤 Exemple de Réponse SOAP

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/">
   <soapenv:Body>
      <CreateDossierResponse>
         <message>Dossier créé avec succès</message>
      </CreateDossierResponse>
   </soapenv:Body>
</soapenv:Envelope>

🛡️ Gestion des Erreurs

  • Référence métier déjà existante
  • Données SOAP invalides
  • Connexion PostgreSQL indisponible
  • XML mal formé
  • Erreur interne du serveur
